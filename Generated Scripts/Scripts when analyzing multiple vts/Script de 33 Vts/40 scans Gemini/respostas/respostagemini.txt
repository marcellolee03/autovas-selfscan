Detalhes da Análise por Vulnerabilidade
Proxmox VE EOL: A maioria dos scripts apenas exibe um aviso, pois uma atualização de sistema completa é muito complexa e arriscada para ser totalmente automatizada. Alguns tentam executar apt dist-upgrade, o que é uma tentativa válida, mas ainda requer supervisão manual.





jQuery XSS: O sucesso desta correção depende muito de onde os scripts assumem que os arquivos vulneráveis do jQuery estão localizados. Scripts que assumem um caminho padrão como /var/www/html/ e substituem os arquivos são eficazes. Scripts que tentam usar wget ou curl contra um IP remoto para substituir um arquivo, sem um mecanismo de upload, são fundamentalmente falhos.





Algoritmos Fracos de SSH: A abordagem mais robusta, adotada por scripts melhores, é substituir as linhas de configuração de Ciphers, KEX e MACs por uma lista de algoritmos fortes conhecidos. Scripts que apenas tentam remover substrings (ex: sed -i '/KexAlgorithms/s/diffie-hellman-group1-sha1//g' ) são frágeis e podem não funcionar se a linha de configuração não existir ou estiver formatada de forma diferente.




Enumeração DCE/RPC: A solução correta é configurar um firewall. Scripts que aplicam uma regra diretamente usando ufw  ou iptables  são eficazes. A maioria, no entanto, apenas exibe um aviso ou um exemplo de regra comentado.





Login Telnet não Criptografado: Esta é a vulnerabilidade mais consistentemente corrigida. Quase todos os scripts usam systemctl para desativar e parar o serviço Telnet e/ou apt para removê-lo.



TLS Deprecado (v1.0/v1.1): Esta correção é específica do serviço (neste caso, RDP na porta 3389). Nenhum script pode corrigir isso de forma confiável em um host Linux. A maioria apenas exibe um aviso. Alguns tentam aplicar configurações genéricas para OpenSSL, o que pode não ter efeito sobre o serviço RDP.





Divulgação de Timestamps TCP/ICMP: A maioria dos scripts corrige essas vulnerabilidades de forma eficaz, adicionando as configurações apropriadas (net.ipv4.tcp_timestamps = 0 e net.ipv4.icmp_echo_ignore_all = 1) ao arquivo /etc/sysctl.conf e aplicando-as com sysctl -p.
